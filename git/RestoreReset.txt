# Discard unstaged changes (working directory)
git restore <file> 
# Unstage a file (remove it from staging area, but keep working directory)
git restore --staged <file>
git reset <file>
# Move HEAD to a previous commit
git reset --soft commit #Move HEAD only, keep staging and working directory.
git reset --hard commit #Reset everything to that commit.
git reset --mixed commit #(default)Move HEAD + reset staging area, keep working directory.
# Recover old file from an old commit
git restore --source <commit> -- <file>
git checkout <commit> -- <file> # legacy way
git log -- <file> # view file history

‚úÖ When to Use Which?
- Use git restore when you want to revert specific files in your working directory or unstage them without affecting commits.
- Use git reset when you want to move the HEAD or clear the staging area in a more aggressive way.
